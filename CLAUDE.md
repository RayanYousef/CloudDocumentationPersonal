## What this repository is

A **Docusaurus 3.x documentation site** in `website/`. The site is deployed to GitHub Pages via a GitHub Actions workflow. Real content goes in `website/docs/`; everything else is site infrastructure.

## Commands

All commands run from the `website/` directory:

```bash
cd website
npm install          # or `npm ci` for a clean, lockfile-exact install
npm run start        # local dev server with hot reload at http://localhost:3000
npm run build        # static build into website/build/
npm run serve        # serve the production build locally
npm run clear        # clear the Docusaurus cache (use when build behaves oddly)
```

Requires Node >= 20 (Docusaurus 3.9, React 19). There are no tests or linters configured.

## Docs structure

- `website/docs/` — all documentation pages. The sidebar is **auto-generated** from the folder structure (`sidebars.js` uses `autogenerated`). No manual sidebar entries are needed.
- Docs are served at the site root (`routeBasePath: '/'`). The page with `slug: /` is the homepage.
- Each folder becomes a sidebar category; control its label/position via `_category_.json`.
- Front matter (`title`, `sidebar_position`, `description`) orders pages within a category.
- Pages that import React components must use the `.mdx` extension.

## Custom components

Components live in `website/src/components/`.

### ModelViewer (glTF)

```mdx
import ModelViewer from '@site/src/components/ModelViewer';

<ModelViewer src="/models/cube.gltf" alt="A cube" />
```

### FbxViewer (FBX)

```mdx
import FbxViewer from '@site/src/components/FbxViewer';

<FbxViewer src="/models/fbx/cube.fbx" />
```

Pass `src` as a root-relative path (e.g. `/models/...`). The component resolves `baseUrl` internally. 3D assets live in `website/static/models/`.

### In-browser editor

Source: `website/src/components/editor/` and the page at `website/src/pages/editor.js`. Accessible at `/editor` on the live site.

## Versioning

- `website/versions.json` lists frozen version numbers (e.g. `["1.0.0"]`).
- `website/versioned_docs/version-X.Y.Z/` holds a snapshot of the docs at that version.
- `website/versioned_sidebars/version-X.Y.Z-sidebars.json` holds the matching sidebar config.
- The current working docs (`website/docs/`) are labeled **Latest** and served at the site root.

## CMS configs

- `.pages.yml` — Pages CMS (https://pagescms.org) configuration. Collections: `intro` (single file, rich-text), `guide` (collection, rich-text), `examples` (collection, raw MDX code body).
- `website/static/admin/config.yml` — Sveltia CMS configuration with the same collections. Sign in with a GitHub Personal Access Token.

## Deployment

`.github/workflows/deploy-docs.yml` triggers on pushes to `main` that touch `website/**`. It runs `npm ci` + `npm run build` and publishes `website/build/` to the `gh-pages` branch.

Site config: `url=https://RayanYousef.github.io`, `baseUrl=/CloudDocumentationPersonal/`. Every absolute in-site link must include that base path. `onBrokenLinks: 'throw'` — every internal link must resolve or the build fails.

## Images

Put images in `website/static/img/` and reference them with absolute paths: `/img/your-image.png`.
